Getting Started - After Installation

The OSCA Console

Once you have successfully loaded the OSCA MMC snapin to MMC, you will see something like this:

In the left-hand pane the OSCA node is displayed - it currently has no CAs defined. In the right-hand pane is the list of Actions that are available - the only possible Action is to enter Admin Mode. The central pane is used to display information about certificates and profiles (it will be populated once you have defined a CA).

Actions

Admin Mode

The default configuration for OSCA is to define a 'User Mode' and an 'Admin Mode'. User Mode is used to conduct daily CA operations such as issuing and revoking certificates. Admin Mode is used for maintenance operations, such as key backups and CA creation.

Entry into admin mode requires a password - this is the password you set during the installaton of the snapin. You can change this password from within Admin Mode
